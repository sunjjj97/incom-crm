# 로컬 데이터를 Railway로 마이그레이션하는 방법

## ⚠️ 중요
로컬(SQLite)과 Railway(PostgreSQL)는 **완전히 다른 데이터베이스**입니다.
따라서 로컬에 등록한 팀, 팀원, 매출 데이터는 **자동으로 Railway로 옮겨지지 않습니다**.

데이터를 Railway로 옮기려면 아래 단계를 따라주세요.

---

## 1단계: 로컬에서 데이터 내보내기

### 방법 1: export_data.py 스크립트 사용 (권장)

로컬 환경에서 다음 명령어를 실행하세요:

```bash
python export_data.py
```

이 스크립트는 `exported_data/` 폴더에 다음 파일들을 생성합니다:
- `auth_user.json` - 관리자 계정 정보
- `crm_app_Department.json` - 사업부 데이터
- `crm_app_Team.json` - 팀 데이터
- `crm_app_TeamMember.json` - 팀원 데이터
- `crm_app_ContractCompany.json` - 계약업체/매출 데이터

### 방법 2: Django 관리 명령어 직접 사용

```bash
# 사용자 데이터
python manage.py dumpdata auth.user --indent 2 --output exported_data/auth_user.json --natural-foreign --natural-primary

# 사업부
python manage.py dumpdata crm_app.Department --indent 2 --output exported_data/crm_app_Department.json --natural-foreign --natural-primary

# 팀
python manage.py dumpdata crm_app.Team --indent 2 --output exported_data/crm_app_Team.json --natural-foreign --natural-primary

# 팀원
python manage.py dumpdata crm_app.TeamMember --indent 2 --output exported_data/crm_app_TeamMember.json --natural-foreign --natural-primary

# 계약업체
python manage.py dumpdata crm_app.ContractCompany --indent 2 --output exported_data/crm_app_ContractCompany.json --natural-foreign --natural-primary
```

---

## 2단계: Railway에 데이터 업로드

### 방법 1: Railway CLI 사용 (권장)

1. Railway CLI 설치:
   ```bash
   npm i -g @railway/cli
   railway login
   ```

2. 프로젝트에 연결:
   ```bash
   railway link
   ```

3. `exported_data/` 폴더를 Railway에 업로드 (또는 개별 파일 업로드):
   ```bash
   railway up exported_data/
   ```

4. Railway에서 load_data.py 실행:
   ```bash
   railway run python load_data.py
   ```

### 방법 2: Railway Web 대시보드 사용

1. Railway 대시보드 → 프로젝트 → "web" 서비스 → "Connect" 탭
2. 터미널이 열리면 다음 명령어 실행:
   ```bash
   # 먼저 exported_data 폴더를 생성
   mkdir -p exported_data
   ```
3. JSON 파일들을 하나씩 복사-붙여넣기 하거나, GitHub에 커밋하여 자동 배포된 후 접근

### 방법 3: GitHub를 통한 업로드 (임시)

⚠️ **주의**: 민감한 데이터(비밀번호 해시 등)가 포함될 수 있으므로, 업로드 후 즉시 삭제하세요.

1. `exported_data/` 폴더를 임시로 Git에 추가 (`.gitignore`에서 제외 해제)
2. GitHub에 푸시
3. Railway에서 자동으로 다운로드됨
4. Railway에서 `load_data.py` 실행
5. GitHub에서 `exported_data/` 폴더 삭제 및 `.gitignore`에 다시 추가

---

## 3단계: Railway에서 데이터 가져오기

### Railway 터미널 접속

1. Railway 대시보드 → 프로젝트 → "web" 서비스
2. **"Connect"** 탭 클릭 (또는 "Deployments" → 최신 배포 → "View logs" 옆에 터미널 아이콘)

### 데이터 로드

```bash
# exported_data 폴더가 있는지 확인
ls exported_data/

# load_data.py 실행
python load_data.py
```

또는 개별 파일 로드:
```bash
python manage.py loaddata exported_data/auth_user.json
python manage.py loaddata exported_data/crm_app_Department.json
python manage.py loaddata exported_data/crm_app_Team.json
python manage.py loaddata exported_data/crm_app_TeamMember.json
python manage.py loaddata exported_data/crm_app_ContractCompany.json
```

## 4단계: 확인

Railway 웹사이트에 접속하여 다음을 확인하세요:
- ✅ 관리자 계정으로 로그인 가능
- ✅ 사업부 목록이 표시됨
- ✅ 팀 목록이 표시됨
- ✅ 팀원 목록이 표시됨
- ✅ 매출 데이터가 표시됨

---

## ⚠️ 주의사항

1. **데이터 순서**: 외래키 관계 때문에 데이터를 로드할 때 순서가 중요합니다:
   - 사용자 (auth.user) → 사업부 → 팀 → 팀원 → 계약업체

2. **중복 데이터**: 이미 데이터가 있는 경우 오류가 발생할 수 있습니다. 이 경우:
   ```bash
   # 기존 데이터 삭제 후 다시 로드 (주의: 모든 데이터가 삭제됩니다!)
   python manage.py flush --noinput
   python load_data.py
   ```

3. **비밀번호 해시**: 사용자 비밀번호는 해시로 저장되므로 그대로 옮겨집니다. 기존 비밀번호로 로그인할 수 있습니다.

4. **ID 충돌**: 데이터베이스가 달라도 Django의 `natural_foreign`, `natural_primary` 옵션을 사용하면 ID 충돌 없이 데이터를 옮길 수 있습니다.

---

## 대안: 수동으로 다시 입력

데이터가 많지 않다면, Railway에서 직접 다시 등록하는 것도 방법입니다:
1. Railway 웹사이트에 접속
2. 관리자 계정 생성 (`python manage.py createsuperuser` 또는 Railway 터미널)
3. 웹 인터페이스에서 사업부, 팀, 팀원, 매출 데이터를 다시 입력

이 방법은 데이터가 적을 때 더 간단할 수 있습니다.
